## Что это за проект

**Полнофункциональный шаблон-генератор** fullstack-приложений на TypeScript под экосистему **biorate**. Репозиторий сам по себе — **не** готовое веб-приложение: в корне нет `apps/` и типичных скриптов сборки продукта. Здесь лежат:

- **Hygen**-шаблоны (`_templates/app/create/…`) — EJS-файлы, из которых при выполнении генератора создаётся монорепозиторий с клиентом и/или сервером.
- **Документация** (`doc/`) — описание архитектуры, конфигурации, коннекторов к БД, брокерам и т.д. (ориентирована на **сгенерированное** приложение, но полезна и при правке шаблонов).
- Вспомогательные скрипты (например `.scripts/renumerate-templates.js`) и конфиг **`.hygen.js`** (хелперы для шаблонов: `server`, `client`, `hygen`, валидация обязательных полей).

Сгенерированный стек (см. `README.md` и `doc/`):

| Слой | Технологии                                                                                                                            |
|------|---------------------------------------------------------------------------------------------------------------------------------------|
| Сервер | **NestJS**, **Inversify** через **@biorate/inversion**, порты/use-case’ы, **infrastructure** (HTTP, persistence, WebSocket по опции)  |
| Клиент | **React**, **MobX**, **react-router**, **i18next**, стили (в т.ч. styled-components, css/less/scss), сборка **Webpack**, шаблоны **Pug** |
| Монорепо | **pnpm** workspaces, **Lerna**, **Nx** (как в генерируемом `package.json`)                                                            |
| Конфиг | **@biorate/config** (JSON-path, шаблоны `${…}` / `@{…}` / `#{…}`)                                                                      |
| Тесты | Клиент: **Playwright**; сервер: **Mocha** (+ при необходимости отчёты Allure)                                                         |

## Обязательные требования окружения

- **Node.js** ≥ 20  
- **pnpm** ≥ 9.7.1 (в `package.json` также указаны совместимые версии npm/yarn)

## Типичный сценарий: создать приложение из шаблона

Из корня этого репозитория:

```bash
pnpm install
npx hygen app help
npx hygen app create --client --server --hygen --app-name=name-of-the-app --app-description='My app name' --add-web-socket --root=tmp
```

Параметры генератора (см. `_templates/app/help/index.ejs.t`):

| Флаг | Назначение |
|------|------------|
| `--client` | Сгенерировать клиентское приложение |
| `--server` | Сгенерировать серверное приложение |
| `--hygen` | Оставить в сгенерированном репо шаблоны Hygen для дальнейшей генерации |
| `--app-name` | Имя продукта (используется в именах пакетов: `{app-name}-client`, `{app-name}-server`) |
| `--app-description` | Краткое описание (в т.ч. отображение во вкладке браузера) |
| `--cut-examples` | Убрать примеры из сгенерированного кода |
| `--add-web-socket` | Добавить WebSocket (Nest gateways + события в шаблоне) |
| `--root` | Папка для создания проекта, можно создавать проект для теста в tmp папке в корне этого шаблона для тестов, но обязательно БЕЗ флага --cut-examples |

Обязательны **`--app-name`** и **`--app-description`** (проверка в `_templates/app/create/00000.00_config.ejs.t` через `h.defined`).

После генерации рабочая разработка идёт **уже в созданном монорепозитории** (команды из генерируемого README), например:

- `pnpm run build:debug` — dev-сборка клиента  
- `pnpm run start:debug` — dev-запуск сервера  

## Где что лежит в этом репозитории

| Путь | Содержание |
|------|------------|
| `_templates/app/create/` | Файлы шаблонов Hygen; имена вида `NNNNN.00_…` задают **порядок** применения |
| `_templates/app/help/` | Текст справки для `hygen app help` |
| `.hygen.js` | Глобальные хелперы EJS (`server`/`client`/`hygen` — условные блоки в шаблонах) |
| `.scripts/renumerate-templates.js` | Перенумерация префиксов файлов в папке шаблонов (после добавления/удаления шагов) |
| `doc/` | Подробная документация по архитектуре и инфраструктуре |

## Навигация по документации (`doc/`)

Начать стоит с:

- `ARCHITECTURE.md` — IoC, **@biorate/inversion**, `injectable` / `init` / `kill`  
- `APP_STRUCTURE.md` — структура монорепо, клиента и сервера  
- `CONFIGURATION.md` и вложенные `ENV_LOADER.md`, `FILE_LOADER.md`, `VAULT_LOADER.md`  
- `CLIENT_APP_ARCH.md` — клиент (MobX-store, inversion)  
- `CONNECTORS.md` и профильные файлы (Sequelize, Kafka, Redis, …) — по мере необходимости  

## Рекомендации для агентов при изменении шаблонов

1. **Понимать два контекста:** правка **генератора** (файлы в `_templates/`) vs правка **уже сгенерированного** приложения в другом каталоге — это разные задачи.  
2. **Порядок файлов в `_templates/app/create/`** важен: префикс `00001`, `00002`, … определяет последовательность. При вставке новых шагов может понадобиться `.scripts/renumerate-templates.js`.  
3. Шаблоны — **EJS**: переменные вроде `APP_NAME`, `SERVER_NAME`, `CLIENT_NAME`, флаги `CLIENT`/`SERVER`/`HYGEN` задаются в `00000.00_config.ejs.t`.  
4. Условная генерация блоков — через хелперы из `.hygen.js` (`h.server(...)`, `h.client(...)`, `h.hygen(...)`) и проверки в EJS.  
5. Не дублировать длинные объяснения в коде: при необходимости расширять `doc/` и ссылаться из README шаблона.  
6. Учитывать ограничение из `APP_STRUCTURE.md`: CI/пайплайн в текущем виде рассчитан на **одну** пару client/server в монорепо.

## Что получает сгенерированный монорепозиторий (как в orbo)

При `hygen app create` в создаваемый **корень** проекта кладётся **`AGENTS.md`** с глобальными правилами (границы приложений, ошибки, тонкие контроллеры/UI, ссылки). Шаблон: `_templates/app/create/00013.01.00_root_AGENTS.md.ejs.t`.

Если включён **`--server`**, дополнительно создаётся **`apps/<app-name>-server/AGENTS.md`** с правилами сервера (слои, DI, статика, тесты, запреты). Шаблон: `_templates/app/create/00085.01.00_apps_server_AGENTS.md.ejs.t`.

Подробности по стеку и слоям сервера — в этих файлах у потребителя шаблона; здесь, в репозитории генератора, достаточно не дублировать их и при правке шаблонов **синхронизировать** оба `AGENTS` с целевой архитектурой (ориентир — референсный монорепозиторий orbo и каталог `doc/`).

Клиент в генерируемом приложении — **MobX** и **@biorate/inversion**, как на сервере.
